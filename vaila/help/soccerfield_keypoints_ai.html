soccerfield_keypoints_ai — Soccer-pitch keypoints (AI seed)

Module: vaila/soccerfield_keypoints_ai.py
GUI button: Field KPs (AI) in Frame B (analysis area, bottom row)
CLI: uv run python -m vaila.soccerfield_keypoints_ai
Default model: vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt (Pose mAP50 = 0.945, trained 400 epochs, recipe-A; April 2026)

What it does

Detects up to 32 soccer-field keypoints (corners, penalty areas, centre circle, penalty spots, …) in broadcast video using a YOLO-pose model. Outputs are CSVs ready for getpixelvideo.py, soccerfield_calib.py (DLT2D homography) and fifa_to_dlt.py (per-frame DLT for rec2d.py/rec3d.py).

Step-by-step (GUI)

  1. Open vailá and click Field KPs (AI) in Frame B.
  2. Pick the broadcast video.
  3. Pick an output folder (vailá creates processed_field_kps_<timestamp>/).
  4. Set:
    • Mode → video
    • Backend → ultralytics
    • Stride → 1 (every frame) or 5 (smoke test)
    • Confidence → 0.30, Draw/save min conf → 0.40
    • imgsz → 1280, Device → 0 for CUDA, empty for auto
    • Ultralytics weights → empty (use bundled) or point to your best.pt
    • Tick Write overlay .mp4
  5. Click Run. The success messagebox shows the output folder.

CLI quick recipes

Recipe 1 — Smoke test (60 frames, default model)

uv run python -m vaila.soccerfield_keypoints_ai \
  --mode video \
  -i tests/sport_fields/ENG_FRA_220243.mp4 \
  -o tests/sport_fields/runs/pitch_kps \
  --backend ultralytics \
  --weights vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt \
  --imgsz 1280 --conf 0.30 --draw-min-conf 0.40 \
  --device 0 --stride 5 --max-frames 60 --overlay-video

Recipe 2 — Full clip (every frame)

uv run python -m vaila.soccerfield_keypoints_ai \
  --mode video -i path/to/video.mp4 -o path/to/output \
  --backend ultralytics \
  --weights vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt \
  --imgsz 1280 --conf 0.30 --draw-min-conf 0.40 \
  --device 0 --stride 1 --overlay-video

Recipe 3 — Batch over a folder

for f in /path/to/videos/*.mp4; do
  uv run python -m vaila.soccerfield_keypoints_ai \
    --mode video -i "$f" -o /path/to/outputs_pitch_kps_full \
    --backend ultralytics \
    --weights vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt \
    --imgsz 1280 --conf 0.30 --draw-min-conf 0.40 \
    --device 0 --stride 1 --overlay-video
done

Recipe 4 — Roboflow (no local weights)

export ROBOFLOW_API_KEY="YOUR_KEY"
uv run python -m vaila.soccerfield_keypoints_ai \
  --mode video -i video.mp4 -o out/ \
  --backend roboflow \
  --roboflow-model-id football-field-detection-f07vi/14 \
  --conf 0.3 --draw-min-conf 0.05 \
  --stride 1 --overlay-video

Recipe 5 — Single frame sanity check

uv run python -m vaila.soccerfield_keypoints_ai \
  --mode frame -i video.mp4 -o out/ --backend ultralytics \
  --weights vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt \
  --frame 0 --imgsz 1280 --conf 0.20 --draw-min-conf 0.20

Training a new detector

Option A — HF sample inside the repo (quick experiments)

Crucial recipe to avoid pose-collapse:

uv run yolo pose train \
  model=yolo26s-pose.pt \
  data=vaila/models/hf_datasets/football-pitch-detection/data/data.yaml \
  epochs=400 imgsz=1280 batch=8 \
  mosaic=0.0 mixup=0.0 close_mosaic=0 erasing=0.0 \
  pose=25.0 kobj=2.0 device=0 patience=80 \
  project=vaila/models/runs/pose_fifa name=pitch32_recipeA_400ep

Option B — External merged dataset (FIFA / multi-source)

Use an absolute data= path to unified/data.yaml produced by vaila.fifa_dataset_builder (dataset root outside the vailá clone). After QA on check_all_labels/, run vaila.fifa_check_labels_dedupe and vaila.fifa_dataset_train_readiness --prune-unified-to-flat --apply-prune so unified/ matches what you validated — see docs/fifa_workflow.md §4.5.

uv run yolo pose train \
  model=yolo11x-pose.pt \
  data=/path/to/dataset_vaila_fifa/unified/data.yaml \
  epochs=200 imgsz=1280 batch=8 \
  mosaic=0 erasing=0 pose=20 kobj=2.5 flipud=0 fliplr=0.5 \
  project=/path/to/dataset_vaila_fifa/runs name=fifa32_retrain \
  device=0
Do not enable mosaic or mixup with small imgsz. It collapses all 32 keypoints onto a single point even while box mAP stays near 1.0 (.claude/skills/soccer-field-keypoints-yolo/SKILL.md).

Roboflow backend missingOptional deps uv add inference supervision + ROBOFLOW_API_KEY
GPU OOMimgsz high or GPU busy --imgsz 960 or --device cpu
