1. Overview

getpixelvideo.py is a comprehensive video annotation tool that allows users to mark and save pixel coordinates in video frames. The tool supports multiple CSV formats and provides advanced visualization capabilities for different types of motion analysis data.

Key Features:

  • Multi-format Support: Load and visualize MediaPipe, YOLO tracking, vailá standard formats, and markerless 2D named-landmark CSVs (frame_index,nose_x,nose_y,nose_z,...)
  • Advanced Visualization: Stick figures for MediaPipe, bounding boxes for YOLO/SAM tracking with colored-ID, ID-only, or ID+confidence overlay modes
  • Flexible Marking: Multiple marker modes for different annotation needs
  • Del Range: Delete one or more marker/keypoint numbers across an inclusive frame range without changing other markers; use commas (0,3,7) or ranges (1:10)
  • Swap Range: Swap paired marker/keypoint lists across an inclusive frame range; first marker line maps pairwise to the second (26,28 with 27,29, or 1:10 with 11:20)
  • Zoom & Navigation: Full zoom capabilities with frame-by-frame navigation
  • Persistence Mode: View marker trails across multiple frames
  • Auto-detection: Automatically detect CSV format or manual selection
  • YOLO-pose export (F9): Turn marker clicks into an Ultralytics pose dataset; reuse kpt_shape when appending to an existing folder (F7)
  • Save ML (button / Ctrl+E): Export a PNG pose dataset with user-selected train/val/test split and build an all_labels/ didactic view with split-prefixed label copies
  • Guide (G): Visual-only overlay for active template (FIFA field or pose skeleton). V toggles the upper-right guide map — does not change how you click; use TAB, Ctrl+G (Go KP), and left click as usual
  • FIFA mode: K or FIFA loads the TOML; Ctrl+E / Save ML and F9 for pose export; backups under .vaila_markers_history/
  • Playback Speed: ] doubles speed (up to 16×); [ halves speed (down to 0.0625×). Current speed shown top-right.

2. Installation & Requirements

Required Dependencies:

  • Python 3.12+
  • OpenCV (cv2): Video processing
  • Pygame: GUI and visualization
  • Pandas: CSV data handling
  • NumPy: Numerical operations
  • Tkinter: File dialogs (usually included with Python)

Installation (vailá repo, recommended):

uv sync
uv run vaila/getpixelvideo.py

CLI Commands:

# Open a video file (GUI opens automatically)
uv run vaila/getpixelvideo.py -f video.mp4

# Open a single PNG image
uv run vaila/getpixelvideo.py -f frame.png

# Open a folder of PNG frames (sequence)
uv run vaila/getpixelvideo.py --sequence /path/to/png_folder/

# Start with pre-loaded keypoint CSV
uv run vaila/getpixelvideo.py -f video.mp4 -k markers.csv

# Start in Labeling (bounding-box) mode
uv run vaila/getpixelvideo.py -f video.mp4 -l

# Open with a pose dataset folder so F9 appends
uv run vaila/getpixelvideo.py -f video2.mp4 --dataset vaila_dataset_20260424_120000/

# Pitch Guide workflow (multi-video FIFA keypoints)
uv run vaila/getpixelvideo.py -f match.mp4
# → click Guide → annotate points → F9 export
uv run vaila/getpixelvideo.py -f match2.mp4 --dataset vaila_dataset_<ts>/
# → Guide → F9 → appends to dataset

Optional one-off (no full uv sync):

uv run --with opencv-python --with pygame --with pandas --with numpy vaila/getpixelvideo.py
Note: On some systems, pygame might require additional system dependencies. If you encounter issues, refer to the pygame installation documentation for your specific platform.

3. Basic Usage

Starting the Application:

uv run vaila/getpixelvideo.py

Optional: --dataset /path/to/dataset (folder with data.yaml) so the next F9 pose export appends; -l starts in labeling mode.

Step-by-Step Process:

  1. Select Video: Choose the video file to process
  2. Load Existing Data: Use 'Load' button in interface (optional)
  3. Select Format: If loading data, choose the CSV format:
    • Auto-detect: Automatically detects the format
    • MediaPipe format: For landmark data with stick figure visualization
    • YOLO tracking format: For tracking data with bounding box visualization
    • vailá standard format: For standard coordinate data
  4. Navigate & Annotate: Use the interface to navigate, zoom, and edit markers; use Del Range to delete one marker/keypoint, comma-separated marker list, or A:B range; use Swap Range to swap paired marker lists/ranges over a frame interval
  5. Save Results: Save the annotated data in CSV format

5. Supported File Formats

Important: The tool loads and displays only x,y coordinates. Z-coordinates and other metadata are ignored during loading.

MediaPipe Format

Used for pose estimation and landmark detection data. The tool extracts x,y coordinates from MediaPipe landmark data.

Input Format: frame_index, nose_x, nose_y, nose_z, left_eye_inner_x, left_eye_inner_y, left_eye_inner_z, ...
Loaded Data: Only x,y coordinates are extracted and displayed
Example:
frame_index,nose_x,nose_y,nose_z,left_eye_inner_x,left_eye_inner_y,left_eye_inner_z 0,100.5,200.3,0.0,150.2,250.1,0.0 1,105.2,205.1,0.0,155.3,255.2,0.0

Note: Z-coordinates are ignored during loading

YOLO Tracking Format

Used for object tracking and detection data. The tool calculates center points from bounding boxes.

Input Format: Frame, Tracker ID, Label, X_min, Y_min, X_max, Y_max, Confidence, Color_R, Color_G, Color_B
Loaded Data: Center points calculated from bounding boxes (X_min+X_max)/2, (Y_min+Y_max)/2
Example:
Frame,Tracker ID,Label,X_min,Y_min,X_max,Y_max,Confidence,Color_R,Color_G,Color_B 0,1,person,100,200,200,300,0.9,255,0,0 0,2,person,300,400,400,500,0.8,0,255,0

vailá Standard Format

Standard coordinate format used by vailá toolbox.

Format: frame, p1_x, p1_y, p2_x, p2_y, p3_x, p3_y, ...
Example:
frame,p1_x,p1_y,p2_x,p2_y 0,100.5,200.3,150.2,250.1 1,105.2,205.1,155.3,255.2

6. Marker Modes

Normal Mode (Default)

The standard marking mode where clicking selects and updates the current marker. Use TAB to navigate between markers. Each marker keeps its ID across all frames. Use Del Range to hide/delete one marker ID, multiple comma-separated IDs, or a sequential A:B range, over an inclusive frame range while preserving other markers. Use Swap Range to exchange paired marker IDs over an inclusive frame range; 26,28 with 27,29 swaps 26<->27 and 28<->29.

  • Use case: Tracking specific points across frames
  • Navigation: TAB to cycle through markers
  • Selection: Click to select and update current marker

1 Line Mode (C key)

Creates points in sequence in one frame. Each click adds a new marker. Ideal for tracing paths or outlines.

  • Use case: Tracing contours, paths, or outlines
  • Activation: Press C key to toggle
  • Behavior: Each click creates a new sequential marker

Sequential Mode (S or O key)

Only in Normal mode. First click fills the currently selected keypoint slot (choose with TAB / SHIFT+TAB or Ctrl+G / Go KP) if that slot is still empty — including keypoint 0. Next clicks advance to selected + 1, etc.

  • Use case: Fast sparse FIFA-style labeling while jumping between indices
  • Activation: Press S or O to toggle
  • Behavior: Automatic ID increment for each new marker

Labeling Mode (L key)

Draws bounding boxes on video frames to create datasets for object detection training (YOLO/COCO format).

  • Labeling Mode: Create bounding box annotations, with save/load project support (F5/F6)
  • Dataset Export: Export structured datasets (train/val/test) with images and JSON annotations
  • Behavior: Click and drag to draw boxes, press Z to undo last box
  • Export: Save exports structured dataset with train/val/test split

Pose dataset export (YOLO-pose) — F9

For keypoint / pose training (e.g. 32 football pitch points), export marker positions as an Ultralytics-compatible layout.

  • When: At least one frame has markers (normal workflow; labeling mode does not need to be on).
  • F9: Creates vaila_dataset_YYYYMMDD_HHMMSS/ next to the video with splits, classes.txt, and data.yaml including kpt_shape: [Nkp, 3] and flip_idx.
  • Append: F7 (Dataset) → pick existing data.yaml; F8 open next video; annotate; F9 again — filenames prefixed with <video_stem>_. Existing kpt_shape is preserved when present.
  • BBox: First user-drawn bbox in labeling mode if any; else tight box around visible keypoints (with padding).
  • Project JSON: <video_stem>_pose_project.json stores raw markers for re-editing.

6b. Guide Mode (Soccer Field / Pose Skeleton)

Guide Mode is a visual-only overlay to help you place keypoints. YOLO-pose training dataset from soccer field keypoints. It loads the field reference CSV (soccerfield_ref3d_fifa.csv or the standard soccerfield_ref3d.csv) and walks you through each point one by one, so you never have to guess which marker to place next.

How to use — GUI:

  1. Open a video: uv run vaila/getpixelvideo.py
  2. Navigate to the frame where the field is clearly visible.
  3. Click the Guide button in the bottom bar (turns orange).
  4. A green panel appears in the top-left corner showing:
    Mark point N/37: (M remaining)
    pN: <point_name>
    The guide also opens with a larger FIFA field reference image showing the field lines and point numbers, with the current point highlighted in yellow.
  5. Left-click on the video → marks the current FIFA point ID and advances to the next.
  6. A → advances/skips the current point (records a None placeholder).
  7. Right-click → deletes the selected/current guide point and returns the guide to that point.
  8. B or Backspace → moves back one point without deleting the marker, useful for correcting a previous click.
  9. G toggles Guide mode without using the mouse.
  10. V shows or hides the FIFA field reference image (FIFA template only).
  11. Repeat until all 37 points are done (guide exits automatically).
  12. Press Save button or S to save the CSV.
  13. Press F9 to export a YOLO-pose dataset from the recorded markers. The export keeps the FIFA order as p1..p37 and writes keypoints.json plus kpt_names in data.yaml.

CLI workflow (multi-video dataset):

# 1. Open first video and create dataset folder
uv run vaila/getpixelvideo.py -f video1.mp4
# → activate Guide, annotate, press F9 → vaila_dataset_YYYYMMDD/

# 2. Open second video appending to the same dataset
uv run vaila/getpixelvideo.py -f video2.mp4 --dataset vaila_dataset_YYYYMMDD/
# → activate Guide, annotate, press F9 → appends to existing dataset

# OR use F8 inside the app to switch videos without closing

Field reference files:

  • vaila/models/soccerfield_ref3d_fifa.csv — FIFA Skeletal Tracking origin (0,0,0 = centre). 37 points. Legacy / 1-based numbering.
  • vaila/models/soccerfield_ref3d.csv — standard vailá layout (origin = bottom-left corner). 37 points. Fallback.
  • vaila/models/soccerfield_ref3d_fifa_dataset.csv — NEW (v0.6.0): 32 keypoints, idx 0 = top_left_corner, with flip_idx. Matches <dataset_root>/unified/keypoints_reference_drawsportsfields.csv and is used by FIFA Labeling Mode.

Legacy PitchGuide (G key) uses the legacy FIFA / standard CSV; Template:FIFA (toolbar Template button or --fifa-dataset) loads the dataset CSV.

6c. FIFA Labeling Mode (Expand the FIFA Dataset)

A focused workflow to append new videos directly to the FIFA dataset at <dataset_root>, ready for retraining the YOLO pitch keypoints model. The on-screen guide shows the FIFA 32-keypoint pitch with idx 0 = top_left_corner through idx 31 = center_circle_right; numbers and order match unified/keypoints_reference_drawsportsfields.csv.

Why a separate mode?

  • The legacy PitchGuide uses 37 points starting at p1 = bottom_left_corner; the FIFA dataset uses 32 points starting at idx 0 = top_left_corner.
  • The FIFA dataset stores files as images/{train,val,test} + labels/{train,val,test} (different from the default train/images layout), and uses names: { 0: football_pitch }.
  • Each keypoint has a fixed flip_idx for horizontal-flip augmentation — these are loaded from the new reference CSV and written into data.yaml automatically.

Activate (three ways)

  1. CLI (recommended):
    uv run vaila/getpixelvideo.py \
      -f /path/to/new_video.mp4 \
      --fifa-dataset <dataset_root>/unified
    Short alias --fifa enables the mode without an existing dataset (F9 will create a fresh vaila_dataset_YYYYMMDD_HHMMSS/ in the FIFA layout).
  2. GUI button: click FIFA in the bottom button cluster. The button turns blue, the 32-kp guide loads, Guide is auto-enabled and the class is set to football_pitch. Click Dataset next to point at unified/data.yaml for append.
  3. Mid-session toggle: click FIFA at any time to switch between the 32-kp dataset reference and the legacy 37-kp guide.

New: FIFA KP button (fixed sparse CSV matrix)

  • Only active when FIFA mode is ON.
  • Left-click: cycle fixed keypoint count N (8, 16, 24, 32, 40, 50, 64).
  • Right-click: toggle header base between 0 and 1.
  • Middle-click: increment start to skip initial keypoints in CSV output.
  • Saving markers (S / Save button) writes a fixed matrix and keeps unmarked keypoints as empty cells.

Workflow per video

  1. The FIFA reference panel shows on the right of the video, with the current keypoint highlighted in yellow.
  2. Click the corresponding pixel — the marker is stored at the FIFA index. The guide advances to the next point automatically.
  3. Skip occluded points with A, step back with B/Backspace, delete with right-click or DELETE.
  4. Navigate frames (Space, ←/→, slider). Pick frames with clear pitch geometry.
  5. Press F9 — new images and labels are appended to unified/{images,labels}/{train,val,test}/ with filename prefix <video_stem>_.

Output layout (FIFA dataset)

<dataset_dir>/
  data.yaml # path: <abs>
              # train: images/train
              # val: images/val
              # test: images/test
              # kpt_shape: [32, 3]
              # flip_idx: [24, 25, ..., 31, 30]
              # names: { 0: football_pitch }
  images/{train,val,test}/<videoname>_frame_NNNNNN.jpg # F9 quick export
  images/{train,val,test}/<videoname>_frame_NNNNNN.png # Save ML export
  labels/{train,val,test}/<videoname>_frame_NNNNNN.txt

Batch over many new videos

for v in /path/to/new_videos/*.mp4; do
    uv run vaila/getpixelvideo.py -f "$v" \
        --fifa-dataset <dataset_root>/unified
done

Or, in the same session: after F9, press F8 to open another video while keeping the dataset target.

Train YOLO with the augmented dataset

uv run yolo pose train \
  data=<dataset_root>/unified/data.yaml \
  model=yolo11x-pose.pt imgsz=1280 mosaic=0 erasing=0 epochs=80
Notes & safeguards:
  • Both layouts (<dir>/{split}/{images,labels} and <dir>/{images,labels}/{split}) are auto-detected when appending — vailá won't create a wrong subdirectory inside an existing FIFA dataset.
  • If vaila/models/soccerfield_ref3d_fifa_dataset.csv is missing, FIFA Mode is auto-disabled and a status message is shown.
  • The legacy 37-kp guide (G key) remains available for non-dataset projects.

7. Data Visualization

MediaPipe Visualization

When loading MediaPipe data, the tool displays stick figures connecting the landmarks:

  • Landmarks: Red dots for each detected point
  • Connections: Green lines connecting related landmarks
  • Pose Structure: Full body pose with head, arms, torso, and legs

YOLO Tracking Visualization

When loading YOLO or SAM tracking data, the tool displays bounding boxes. The loader asks for colored boxes only, ID only, or ID + confidence. Save exports loaded tracking bboxes as a YOLO detection dataset when Labeling mode is not active:

  • Bounding Boxes: Colored rectangles around detected objects
  • Labels: Object class and tracker ID displayed
  • Colors: Unique colors for different tracker IDs
  • Confidence: Detection confidence values shown

SAM3 boxes → player DETECTION dataset (recommended flow)

  1. Open the video, then Load Track CSV and pick sam_tracks.csv.
  2. "BBox display" prompt: pick any (e.g. 3 = ID+confidence) — visual only.
  3. "Convert boxes to markers?" prompt: press Enter to SKIP — this KEEPS the boxes (correct for per-player tracking). Choosing an anchor (1–5) turns each player into a keypoint of ONE object (a POSE dataset that cannot detect players individually; yolotrain will block it).
  4. Save → writes a YOLO detection dataset (data.yaml, classes.txt, train/val/test, one box per player per frame).

Temporal split (default since v0.3.65): detection and pose exports split frames into chronological train/val/test blocks (first → train, middle → val, last → test) instead of a random shuffle, so near-duplicate consecutive video frames do not leak across train/val and inflate mAP. The terminal prints split=temporal (or split=random).

Equivalent CLI (no GUI):

uv run python -m vaila.sam_to_yolo build \
  --sam-tracks /ABS/path/sam_tracks.csv \
  --video /ABS/path/video.mp4 \
  --class-name person          # --split-mode temporal is the default

vailá Standard Visualization

Standard point marker visualization:

  • Markers: Green circles for each point
  • Numbers: Marker IDs displayed next to points
  • Selection: Orange highlight for selected marker

8. Saving Options

Standard Save (S key)

Saves current markers in the standard vailá format.

  • Format: frame, p1_x, p1_y, p2_x, p2_y, ...
  • File: {video_name}_markers.csv
  • Location: Same directory as video file

In FIFA mode, if configured with FIFA KP, Save uses a fixed sparse matrix based on N, start, and header base (0 or 1).

Example (FIFA mode, N=8, start=5, base=0):
frame,p5_x,p5_y,p6_x,p6_y,p7_x,p7_y,p8_x,p8_y,p9_x,p9_y,p10_x,p10_y,p11_x,p11_y,p12_x,p12_y 120,,,,1020.4,410.1,1005.8,398.7,,,,,,,,, 121,,,,1018.6,409.2,1003.5,397.2,,,,,,,,,
Example (same setup, base=1):
frame,p6_x,p6_y,p7_x,p7_y,p8_x,p8_y,p9_x,p9_y,p10_x,p10_y,p11_x,p11_y,p12_x,p12_y,p13_x,p13_y 120,,,,1020.4,410.1,1005.8,398.7,,,,,,,,,

1 Line Save

Saves sequential markers from 1 Line mode.

  • Format: frame, p1_x, p1_y, p2_x, p2_y, ...
  • File: {video_name}_markers_sequential.csv
  • Use: For path tracing and contour data

Sequential Save

Saves markers with incrementing IDs.

  • Format: frame, p1_x, p1_y, p2_x, p2_y, ...
  • File: {video_name}_markers_sequential.csv
  • Use: For multiple point annotations

Labeling Save (Bounding Box Mode)

Saves bounding boxes as structured dataset for Machine Learning training.

  • Export: Save exports structured dataset by split
  • Project Management: F5 Save & export dataset, F6 Load project, F7 Load dataset folder (next Save appends), F8 Open another video (keeps dataset)
  • Renaming: Press N to rename object labels
  • Directory: New export creates dataset_YYYYMMDD_HHMMSS/; use F7 to select existing data.yaml to append (multi-video)
  • Structure: train/val/test folders with images/ and labels/
  • Split: Automatic 70/20/10 split (train/val/test)
  • Use: For creating object detection datasets (YOLO/COCO format)
  • Activation: Save when Labeling mode is active (F5 or Save button)

Pose dataset save (YOLO-pose) — F9

  • Format: One Ultralytics pose label line per frame (cls cx cy w h + normalized keypoints with visibility).
  • Directory: New vaila_dataset_YYYYMMDD_HHMMSS/; append via F7 or --dataset before F9.
  • Use: Retrain pose models (e.g. soccer field keypoints).
  • Activation: Press F9 when markers exist.

Save ML dataset (PNG split + all_labels) — Ctrl+E or button

  • Export: Runs split export (train/val/test) using the same base pose flow as F9 and saves frame images as .png.
  • Split presets: choose 1 for classic 70/15/15, 2 for training focus 80/10/10, or 3 for Big Data 98/1/1. You can also type custom percentages, for example 75,15,10.
  • Didactic view: Also writes all_labels/ in dataset root with split-prefixed label copies (train_*, val_*, test_*).
  • Scope: Works for standard and FIFA labeling workflows.

9. Advanced Features

Persistence Mode (P key)

Shows markers from previous frames to visualize movement trails.

  • Activation: Press P to toggle
  • Adjustment: Press 1/2 to decrease/increase persistence frames
  • Full Toggle: Press 3 for full persistence
  • Visualization: Fading trails show marker movement

Labeling Mode (L key)

Create bounding box annotations for object detection model training.

  • Activation: Press L key or click "Labeling" button
  • Usage: Click and drag to draw bounding boxes on video frames
  • Edit: Press Z to remove last box in current frame
  • Export: Save creates structured dataset with train/val/test split
  • Dataset Structure: Images and JSON annotations organized by split
  • Format: JSON files with x, y, w, h coordinates and labels
  • Save Project: F5 to save and export dataset (creates dataset_YYYYMMDD_HHMMSS/ or appends if dataset loaded via F7)
  • Load Project: F6 to load saved state from JSON
  • Load Dataset: F7 to select dataset folder (next F5 appends)
  • Open another video: F8 to switch video without closing the app (keeps dataset; auto-loads project if present)
  • Rename: Press N to rename the label (e.g., "ball", "player")
  • Use case: Creating datasets for YOLO, COCO, or other ML frameworks

Pose dataset (F9)

Summarized: marker-based YOLO-pose export, multi-video via F7/F8, optional --dataset CLI flag. See sections 6 and 8 for full detail.

Auto-detection

The tool automatically detects CSV format based on column structure:

  • MediaPipe: Detects 'landmark' in column names
  • YOLO: Detects 'Frame', 'Tracker ID', 'X_min' columns
  • vailá: Detects 'frame' and 'p' pattern in columns

Backup and Recovery

Built-in backup system for data safety:

  • Auto-backup: Timestamped copies are written automatically before major edits (see make_backup() in the module)
  • Hidden history: Backups are stored in .vaila_markers_history/<video_stem>/
  • Retention: Keeps only the latest 100 backups per video
  • Reload coordinates: Use the Load button and pick your CSV again (there is no dedicated "reload" hotkey — L toggles Labeling mode)

Documentation Access

  • Quick Help: Press H for the in-app overlay
  • HTML guide: Press ? (Help Web) to open vaila/help/getpixelvideo.html in your browser
  • Canonical docs: vaila/help/getpixelvideo.md (Markdown) and vaila/help/getpixelvideo.html (browser)

10. Troubleshooting

Common Issues

Video Not Loading

  • Check if video file is corrupted
  • Ensure video format is supported (MP4, AVI, MOV, MKV)
  • Verify file path doesn't contain special characters

CSV Format Not Detected

  • Check CSV file structure matches expected format
  • Use manual format selection if auto-detection fails
  • Verify CSV file is not corrupted

Performance Issues

  • Reduce video resolution for better performance
  • Close other applications to free memory
  • Use smaller zoom levels for large videos

Visualization Problems

  • Ensure CSV data is in the correct format
  • Check if coordinate values are within video dimensions
  • Verify landmark connections for MediaPipe data

Error Messages

Error Solution
"Error opening video file" Check video format and file integrity
"No keypoint file selected" Select a valid CSV file or start fresh
"Unknown format" Use manual format selection
"Error loading coordinates" Check CSV file format and structure
Support: For additional help, check the vailá project documentation or create an issue on the GitHub repository.
