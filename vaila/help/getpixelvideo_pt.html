1. Visão Geral

getpixelvideo.py é uma ferramenta abrangente de anotação de vídeo que permite aos usuários marcar e salvar coordenadas de pixels em quadros de vídeo. A ferramenta suporta múltiplos formatos CSV e fornece recursos avançados de visualização para diferentes tipos de dados de análise de movimento.

Principais Recursos:

  • Suporte Multi-formato: Carregar e visualizar formatos MediaPipe, YOLO tracking e vailá padrão
  • Visualização Avançada: Figuras de palitos para MediaPipe, caixas delimitadoras para YOLO tracking
  • Marcação Flexível: Múltiplos modos de marcadores para diferentes necessidades de anotação
  • Modo Labeling: Criar anotações de caixas delimitadoras para datasets de Machine Learning
  • Dataset YOLO-pose (F9): Exportar marcadores como dataset Ultralytics pose (data.yaml com kpt_shape, splits train/val/test); anexar vários vídeos com F7 + F8
  • Zoom & Navegação: Capacidades completas de zoom com navegação quadro a quadro
  • Modo Persistência: Visualizar trilhas de marcadores através de múltiplos quadros
  • Auto-detecção: Detectar automaticamente formato CSV ou seleção manual
  • Documentação Abrangente: Arquivo de ajuda HTML com instruções detalhadas

2. Instalação e Requisitos

Dependências Necessárias:

  • Python 3.12+
  • OpenCV (cv2): Processamento de vídeo
  • Pygame: GUI e visualização
  • Pandas: Manipulação de dados CSV
  • NumPy: Operações numéricas
  • Tkinter: Diálogos de arquivo (geralmente incluído com Python)

Instalação:

No repositório vailá (recomendado):

uv sync
uv run vaila/getpixelvideo.py

Dependências avulsas:

pip install opencv-python pygame pandas numpy
Nota: Em alguns sistemas, o pygame pode exigir dependências adicionais do sistema. Se encontrar problemas, consulte a documentação de instalação do pygame para sua plataforma específica.

3. Uso Básico

Iniciando a Aplicação:

uv run vaila/getpixelvideo.py

Processo Passo-a-Passo:

  1. Selecione o Vídeo: Escolha o arquivo de vídeo para processar
  2. Carregue Dados Existentes: Use o botão 'Load' na interface (opcional)
  3. Selecione o Formato: Se carregando dados, escolha o formato CSV:
    • Auto-detectar (recomendado): Detecta automaticamente o formato
    • Formato MediaPipe: Para dados de landmarks com visualização de stick figure
    • Formato YOLO tracking: Para dados de tracking com visualização de bounding box
    • Formato vailá padrão: Para dados de coordenadas padrão
  4. Navegue e Anote: Use a interface para navegar, fazer zoom e editar marcadores (TAB / SHIFT+TAB, Ctrl+G para saltar para um keypoint pelo número; botão Del Range apaga um ou mais marcadores por lista/intervalo; botão Swap Range troca pares de marcadores em um intervalo de quadros)
  5. Salve os Resultados: Salve os dados anotados em formato CSV

5. Formatos de Arquivo Suportados

Importante: A ferramenta carrega e exibe apenas coordenadas x,y. Coordenadas Z e outros metadados são ignorados durante o carregamento.

Formato MediaPipe

Usado para dados de estimação de pose e detecção de landmarks. A ferramenta extrai coordenadas x,y dos dados de landmark do MediaPipe.

Formato de Entrada: frame_index, nose_x, nose_y, nose_z, left_eye_inner_x, left_eye_inner_y, left_eye_inner_z, ...
Dados Carregados: Apenas coordenadas x,y são extraídas e exibidas
Exemplo:
frame_index,nose_x,nose_y,nose_z,left_eye_inner_x,left_eye_inner_y,left_eye_inner_z 0,100.5,200.3,0.0,150.2,250.1,0.0 1,105.2,205.1,0.0,155.3,255.2,0.0

Nota: Coordenadas Z são ignoradas no carregamento

Formato YOLO Tracking

Usado para dados de tracking e detecção de objetos. A ferramenta calcula pontos centrais a partir das caixas delimitadoras.

Formato de Entrada: Frame, Tracker ID, Label, X_min, Y_min, X_max, Y_max, Confidence, Color_R, Color_G, Color_B
Dados Carregados: Pontos centrais calculados das bounding boxes (X_min+X_max)/2, (Y_min+Y_max)/2
Exemplo:
Frame,Tracker ID,Label,X_min,Y_min,X_max,Y_max,Confidence,Color_R,Color_G,Color_B 0,1,person,100,200,200,300,0.9,255,0,0 0,2,person,300,400,400,500,0.8,0,255,0

Formato vailá Padrão

Formato de coordenadas padrão usado pela toolbox vailá.

Formato: frame, p1_x, p1_y, p2_x, p2_y, p3_x, p3_y, ...
Exemplo:
frame,p1_x,p1_y,p2_x,p2_y 0,100.5,200.3,150.2,250.1 1,105.2,205.1,155.3,255.2

6. Modos de Marcação

Modo Normal (Padrão)

O modo de marcação padrão onde clicar seleciona e atualiza o marcador atual. Use TAB para navegar entre marcadores. Cada marcador mantém seu ID em todos os quadros.

  • Caso de uso: Rastrear pontos específicos através dos quadros
  • Navegação: TAB para circular entre marcadores
  • Seleção: Clique para selecionar e atualizar o marcador atual

Modo 1 Line (tecla C)

Cria pontos em sequência em um quadro. Cada clique adiciona um novo marcador. Ideal para traçar caminhos ou contornos.

  • Caso de uso: Traçar contornos, caminhos ou outlines
  • Ativação: Pressione a tecla C para alternar
  • Comportamento: Cada clique cria um novo marcador sequencial

Modo Sequencial (tecla O / S)

Cada clique cria um novo marcador com IDs incrementais. Não é necessário selecionar marcadores primeiro. Disponível apenas no Modo Normal.

  • Caso de uso: Anotação rápida de múltiplos pontos
  • Ativação: Pressione a tecla O ou S para alternar
  • Comportamento: Incremento automático de ID para cada novo marcador

Modo Labeling (tecla L)

Desenha caixas delimitadoras nos quadros de vídeo para criar datasets para treinamento de detecção de objetos (formato YOLO/COCO).

  • Modo Labeling: Crie anotações de bounding box, com suporte para salvar/carregar projeto (F5/F6)
  • Exportação de Dataset: Exporte datasets estruturados (train/val/test) com imagens e anotações JSON
  • Comportamento: Clique e arraste para desenhar caixas, pressione Z para desfazer a última caixa
  • Exportação: Salvar exporta dataset estruturado com divisão train/val/test
  • Multi-vídeo: F7 carrega pasta do dataset; F8 abre outro vídeo sem fechar o app

Exportação de dataset pose (YOLO-pose) — F9

Para gerar dataset de keypoints / pose (ex.: campo de futebol com 32 pontos), não só detecção por bbox:

  • Quando: Com marcadores em pelo menos um quadro (fluxo normal de marcadores; o modo Labeling não precisa estar ativo).
  • F9: Cria pose_dataset_YYYYMMDD_HHMMSS/ ao lado do vídeo com train|val|test/{images,labels}/, classes.txt e data.yaml (inclui kpt_shape: [Nkp, 3] e flip_idx).
  • Multi-vídeo / anexar: F7 escolhe o data.yaml de um dataset existente; F8 abre o próximo vídeo; anote e F9 de novo.
  • CLI: uv run vaila/getpixelvideo.py -f VIDEO.mp4 --dataset /caminho/dataset_pose
  • JSON de projeto: <nome_video>_pose_project.json no dataset guarda os marcadores brutos para reedição.

Modo Pitch Guide (keypoints FIFA do campo)

Fluxo guiado para rotular pontos do campo de futebol em ordem fixa. O modo carrega soccerfield_ref3d_fifa.csv automaticamente e grava os marcadores como p1..p37, preservando a ordem esperada para calibração e treino YOLO-pose.

  • Ativação: clique no botão PitchGuide ou pressione G.
  • Overlay: painel verde mostra o ponto atual e a coordenada de referência.
  • Imagem: uma referência maior do FIFA Field aparece inicialmente com linhas do campo, números dos pontos e o ponto atual em amarelo; pressione V para mostrar/esconder.
  • Marcar: clique esquerdo grava o ponto FIFA atual e avança automaticamente.
  • Pular: A avança o guia e grava o ponto como ausente.
  • Apagar: clique direito apaga o ponto selecionado/atual e retorna o guia para ele.
  • Corrigir: B ou Backspace volta um ponto sem apagar o marcador já salvo.
  • Exportar: F9 gera dataset YOLO-pose com kpt_shape, kpt_names e keypoints.json.
  • TOML FIFA: Tecla K (ou botão FIFA) carrega/cria o TOML ao lado do vídeo (n_keypoints, start, base).

7. Visualização de Dados

Visualização MediaPipe

Ao carregar dados MediaPipe, a ferramenta exibe figuras de palitos conectando os landmarks:

  • Landmarks: Pontos vermelhos para cada ponto detectado
  • Conexões: Linhas verdes conectando landmarks relacionados
  • Estrutura de Pose: Pose corporal completa com cabeça, braços, torso e pernas

Visualização YOLO Tracking

Ao carregar dados YOLO tracking, a ferramenta exibe caixas delimitadoras:

  • Caixas Delimitadoras: Retângulos coloridos ao redor de objetos detectados
  • Labels: Classe do objeto e ID do tracker exibidos
  • Cores: Cores únicas para diferentes IDs de tracker
  • Confiança: Valores de confiança de detecção mostrados

Visualização vailá Padrão

Visualização padrão de marcadores pontuais:

  • Marcadores: Círculos verdes para cada ponto
  • Números: IDs dos marcadores exibidos ao lado dos pontos
  • Seleção: Destaque laranja para marcador selecionado

8. Opções de Salvamento

Salvamento Padrão (tecla S)

Salva marcadores atuais no formato padrão vailá.

  • Formato: frame, p1_x, p1_y, p2_x, p2_y, ...
  • Arquivo: {nome_video}_markers.csv
  • Localização: Mesmo diretório do arquivo de vídeo

Salvamento 1 Line

Salva marcadores sequenciais do modo 1 Line.

  • Formato: frame, p1_x, p1_y, p2_x, p2_y, ...
  • Arquivo: {nome_video}_markers_sequential.csv
  • Uso: Para traçado de caminhos e dados de contorno

Salvamento Sequencial

Salva marcadores com IDs incrementais.

  • Formato: frame, p1_x, p1_y, p2_x, p2_y, ...
  • Arquivo: {nome_video}_markers_sequential.csv
  • Uso: Para anotações de múltiplos pontos

Salvamento Labeling (Modo de Caixas Delimitadoras)

Salva caixas delimitadoras como dataset estruturado para treinamento de Machine Learning.

  • Exportação: Salvar exporta dataset estruturado por divisão (split)
  • Gerenciamento de Projeto: F5 para Salvar, F6 para Carregar projetos de labeling
  • Renomear: Pressione N para renomear labels de objetos
  • Diretório: Novo: dataset_YYYYMMDD_HHMMSS/; reutilizar: F7 → selecione o data.yaml existente → próximo F5 anexa
  • Estrutura: pastas train/val/test com images/ e labels/
  • Divisão: Divisão automática 70/20/10 (train/val/test)
  • Uso: Para criar datasets de detecção de objetos (formato YOLO/COCO)
  • Ativação: Salvar quando o modo Labeling estiver ativo (F5)

Salvamento pose (YOLO-pose) — F9

Exporta dataset de keypoints/pose a partir dos marcadores clicados.

  • Formato: Labels YOLO-pose Ultralytics (.txt por imagem: cls cx cy w h + kp_x kp_y v × Nkp)
  • Diretório: Novo: pose_dataset_YYYYMMDD_HHMMSS/ ao lado do vídeo; anexar: F7 ou --dataset antes do F9
  • Estrutura: Mesmos splits (train/val/test, images/, labels/), mais data.yaml com kpt_shape e names
  • Uso: Retreino de redes de pose (ex.: keypoints do campo)
  • Ativação: F9 com pelo menos um quadro com marcadores

9. Recursos Avançados

Modo Persistência (tecla P)

Mostra marcadores de quadros anteriores para visualizar trilhas de movimento.

  • Ativação: Pressione P para alternar
  • Ajuste: Pressione 1/2 para diminuir/aumentar quadros de persistência
  • Alternância Completa: Pressione 3 para persistência completa
  • Visualização: Trilhas desvanecidas mostram movimento do marcador

Modo Labeling (tecla L)

Crie anotações de bounding box para treinamento de modelos de detecção de objetos.

  • Ativação: Pressione a tecla L ou clique no botão "Labeling"
  • Uso: Clique e arraste para desenhar caixas delimitadoras nos quadros de vídeo
  • Editar: Pressione Z para remover a última caixa no quadro atual
  • Exportação: Salvar cria dataset estruturado com divisão train/val/test
  • Estrutura do Dataset: Imagens e anotações JSON organizadas por divisão
  • Formato: Arquivos JSON com coordenadas x, y, w, h e labels
  • Salvar Projeto: F5 para salvar estado atual em JSON (permite resumir depois)
  • Carregar Projeto: F6 para carregar estado salvo de JSON
  • Renomear: Pressione N para renomear o label (ex: "bola", "jogador")
  • Caso de uso: Criar datasets para YOLO, COCO, ou outros frameworks de ML

Auto-detecção

A ferramenta detecta automaticamente formato CSV baseado na estrutura das colunas:

  • MediaPipe: Detecta 'landmark' nos nomes das colunas
  • YOLO: Detecta colunas 'Frame', 'Tracker ID', 'X_min'
  • vailá: Detecta padrão 'frame' e 'p' nas colunas

Backup e Recuperação

Sistema de backup integrado para segurança dos dados:

  • Backup: Pressione B para criar backup
  • Recarregar: Pressione L para recarregar do arquivo
  • Auto-backup: Backups automáticos antes de operações importantes

Acesso à Documentação

  • Ajuda Rápida: Pressione H para ajuda no aplicativo
  • Documentação Completa: Pressione ? para abrir a documentação HTML no navegador
  • Documentação HTML: vaila/help/getpixelvideo_pt.html (PT) e vaila/help/getpixelvideo.html (EN)

10. Solução de Problemas

Problemas Comuns

Vídeo Não Carrega

  • Verifique se o arquivo de vídeo está corrompido
  • Certifique-se de que o formato de vídeo é suportado (MP4, AVI, MOV, MKV)
  • Verifique se o caminho do arquivo não contém caracteres especiais

Formato CSV Não Detectado

  • Verifique se a estrutura do arquivo CSV corresponde ao formato esperado
  • Use seleção manual de formato se a auto-detecção falhar
  • Verifique se o arquivo CSV não está corrompido

Problemas de Performance

  • Reduza a resolução do vídeo para melhor performance
  • Feche outros aplicativos para liberar memória
  • Use níveis menores de zoom para vídeos grandes

Problemas de Visualização

  • Certifique-se de que os dados CSV estão no formato correto
  • Verifique se os valores de coordenadas estão dentro das dimensões do vídeo
  • Verifique conexões de landmarks para dados MediaPipe

Mensagens de Erro

Erro Solução
Error opening video file Verifique formato do vídeo e integridade do arquivo
No keypoint file selected Selecione um arquivo CSV válido ou comece do zero
Unknown format Use seleção manual de formato
Error loading coordinates Verifique formato e estrutura do arquivo CSV
Suporte: Para ajuda adicional, verifique a documentação do projeto vailá ou crie uma issue no repositório GitHub.
