CLI: Yes

Description

transcribe_pdfs.py converts typed, scanned, or handwritten PDF exams into text files. It writes one .txt and one .report.json per PDF, plus a batch_report.csv summary.

GUI Workflow

  1. Open vaila.py.
  2. Click Brainstorm in Frame B.
  3. Use the separate PDF Transcription section near the top and click Transcribe PDFs.
  4. Select a PDF directory or a single PDF.
  5. Confirm output options and click Transcribe PDFs.
  6. Check the source feedback line for the selected path and PDF count.

Default PAE Folders

~/Preto/USP_RP/Alunos/PAE_USP/PAE_Biomec1/originais
~/Preto/USP_RP/Alunos/PAE_USP/PAE_Biomec1/transcritas_originais

Modes

ModeUse
autoTry native text first; if not useful, use vision transcription.
visionRender pages and send images to Gemini CLI. Best for handwritten exams.
nativeUse embedded PDF text only. Does not require Gemini.
ocrReserved for future local OCR backend.

Requirements

  • pdftotext, pdftoppm, and pdfinfo or qpdf
  • gemini CLI for auto / vision handwritten transcription
  • Python packages used by vailá: Pillow, tkinter, and optional tqdm

CLI Examples

uv run python -m vaila.transcribe_pdfs --gui
uv run python -m vaila.transcribe_pdfs \
  --input-dir ~/Preto/USP_RP/Alunos/PAE_USP/PAE_Biomec1/originais \
  --output-dir ~/Preto/USP_RP/Alunos/PAE_USP/PAE_Biomec1/transcritas_originais \
  --mode auto
Review required: check pages listed in low_confidence_pages and markers such as [ilegível], [revisar], and [palavra incerta: ...] before using the transcription for grading or records.
