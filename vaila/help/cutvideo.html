📖 Description

Interactive video cutting with frame-accurate navigation, TOML-based cut storage, batch/sync workflows, and high-resolution timestamps (.6f) suitable for biomechanics.

Key Updates

  • TOML cuts: Saves/loads {basename}_cuts.toml with 1-based frames, .6f precision, frame_count = end - start + 1, plus output_dir and per-cut output_file. Paths stored as POSIX.
  • Audio waveform: Panel with orange waveform, sync playback; A toggle, M mute.
  • Loop control: Loop button stacked above Help / Base name (right side of control bar).
  • Auto-fit: Hotkey 0; zoom/pan with +/-, wheel, middle-mouse drag.
  • Marker navigation: Shift+Left/Right cycles previous/next cut marker with wraparound, matching getpixelvideo; PageUp/PageDown and Home/End remain available.
  • Cut timeline feedback: clickable/draggable strip above the scrub slider, matching getpixelvideo. Blue = range, green = start, orange = end, yellow = pending start, white = playhead. Marker columns snap to cut points.
  • Responsive final render: ffmpeg/OpenCV export uses a cancellable Tk progress dialog that remains responsive while rendering.
  • Custom output base name: B or Base name button → {prefix}_frame_{start}_to_{end}.mp4; folder still {video}_vailacut_{timestamp}.
  • Per-cut output names (CSV/TXT): V or N or Cut names button — one name per line → name.mp4 per cut (example: CarlosMiguel_cod_02, …). Does not load cuts or replace TOML (F); only exported video filenames. C = labels in TOML only.
  • Subprocess launch: started from the main GUI as a separate process (like getpixelvideo), so closing the cut window returns control to vailá without freezing the main window.
  • Playback speed (v0.3.76): [ / ] step through discrete speeds (0.0625×–16×, always includes 1×); HUD upper-right on the video (same style as getpixelvideo).
  • Player startup fix (v0.3.47): output_basename initialized before pygame UI.

🔧 Main Functions (high level)

  • play_video_with_cuts() — main UI/player and cut workflow.
  • timeline_x_for_frame() / frame_index_from_cut_timeline_x() — strip drawing and marker-aware click mapping.
  • adjacent_cut_marker_frame() — Shift+Left/Right previous/next cut-marker lookup with wraparound.
  • RenderProgressDialog — responsive, cancellable export feedback.
  • sanitize_output_basename(), effective_cut_basename(), cut_output_filename() — output naming.
  • prompt_output_basename_dialog() — Tk dialog for single base name.
  • prompt_output_basenames_csv(), parse_basename_list(), build_cut_output_filenames() — per-cut name list (N / Names CSV) and collision-safe filename resolution.
  • save_cuts_to_toml() — writes TOML (1-based frames, .6f times).
  • load_cuts_from_toml() / load_cuts_from_toml_file() — reads TOML; legacy .txt fallback.
  • cut_video_with_ffmpeg() / cut_video_with_opencv() — cutting backends.
  • batch_process_videos(), batch_process_sync_videos() — batch workflows.
  • get_precise_video_metadata() — ffprobe-first metadata.
  • run_cutvideo() — GUI entry point.

🎮 Controls (UI)

Bottom control bar

  • Left: clickable cut strip above slider, frame/time (.6f), hint Out: {prefix}_frame_… (or Out: per-cut CSV names (n)). Strip: blue range, green start, orange end, yellow pending, white playhead.
  • Right: Loop, Help, then Cut names (Names ✓n when a list is loaded) + Base name (Base ✓ when custom prefix is set).

Keyboard

  • Playback: Space, [ / ] (slower/faster), →/←, Shift+→/← (previous/next cut marker with wraparound), ↑/↓ (±60), slider click, cut-strip click/drag with marker snap.
  • Audio: A panel, M mute.
  • Window: 0 auto-fit; +/- / wheel zoom; MMB pan.
  • Cuts: S, E, R, D/Delete, L, C (labels CSV).
  • Markers: Shift+Left/Right, PageUp/PageDown, Home/End.
  • Jump: G (frame), T (time), I/P (FPS).
  • Files: F (load TOML/sync), V / N (per-cut names), B (single base), C (labels), ESC (save + optional render).
  • Help: H or Help button.

📁 Formats

  • Input videos: .mp4, .avi, .mov, .mkv.
  • Cuts file: {basename}_cuts.toml (1-based frames, output_file per cut).
  • Sync TXT: video_file new_name initial_frame final_frame.
  • Outputs:
    • Folder: {video}_vailacut_{timestamp} (or _sync_vailacut_ in sync mode).
    • Files: {base}_frame_{start}_to_{end}.mp4 (1-based; custom {base} via B).
    • Files (per-cut CSV via N): {name}.mp4, one name per cut, de-duplicated.

🔬 Precision Notes

  • FPS and times use .6f; duration from inclusive frame_count.
  • UI frames are 1-based; internal math is 0-based.

🐛 Troubleshooting Quick Hits

  • Main vailá window froze / needed kill after cutting: Fixed — separate process plus responsive cancellable progress dialog during final export.
  • Player does not open after metadata: Use v0.3.47+ (output_basename init fix).
  • No ffmpeg: OpenCV fallback; install ffmpeg for precise cuts and audio.
  • Cannot load cuts: Use .toml with [[cuts]]; legacy .txt deprecated.
  • No audio: Video may lack audio track; panel shows "No Audio Data".
