vailá - sam_to_yolo

File: vaila/sam_to_yolo.py

Version: 0.3.67 | Updated: 2026-06-29

Description

Converts a SAM 3 video tracking export (sam_tracks.csv or its sam_bbox_tracks.csv alias) into a YOLO detection dataset: one bounding box per tracked instance per frame, a single class (person by default). The per-instance IDs 1..N are assigned by the tracker (BoT-SORT) at inference — not by detector classes.

Why this module exists

A common mistake: run SAM 3 → load tracks into getpixelvideo → convert each bbox to one marker → press F9 (pose export). That collapses all instances of a frame into one object whose keypoints are the markers (nc: 1, names: ['object'], kpt_shape: [N, 3]). A model trained that way predicts one box + N keypoints and never detects N separate people. sam_to_yolo skips that detour.

Label format

0 cx cy w h      # instance 1 (class 0 = person)
0 cx cy w h      # instance 2
...              # ~N lines, one per SAM track in the frame

data.yaml: nc: 1, names: ['person'], no kpt_shape.

Variable number of instances

The detector + tracker do not require a fixed count. Each frame's label holds exactly as many boxes as SAM tracked in that frame (7, 16, 18, ...). Nothing is hard-coded to 16.

Class name is metadata (rename anytime)

YOLO detection labels store only the class index (0), never the text. So object → person is a metadata edit — the thousands of .txt files never change. Pick the name at build time, rename the dataset before training, or rename the trained weights after training.

CLI

Build a detection dataset:

uv run python -m vaila.sam_to_yolo build \
  --sam-tracks /path/processed_sam_*/<video>/sam_tracks.csv \
  --video /path/<video>.mp4 \
  --class-name person \
  --reuse-images-dir /path/vaila_dataset_*   # optional, reuse extracted frames \
  --output /path/out_dataset                 # optional

(A bare --sam-tracks ... with no subcommand still implies build.)

Rename the class of an existing dataset (metadata only; labels untouched):

uv run python -m vaila.sam_to_yolo rename-dataset --dataset /path/out_dataset --class-name person

Rename the class baked into a trained .pt (no retraining):

uv run python -m vaila.sam_to_yolo rename-model --weights best.pt --class-name person --output best_person.pt
  • --split-mode {temporal,random}: temporal is the default — chronological train/val/test blocks (first frames train, middle val, last test). On video a random split leaks near-duplicate consecutive frames across train/val and inflates mAP. Use random only for the old shuffle behaviour.
  • --frame-stride N: keep only every Nth frame (10 keeps 0,10,20,...). 1 (default) = full extract. Larger = smaller, less-redundant labelling set from the SAM3 tracks. A banner + tqdm bar show progress; extraction is a single sequential decode pass (much faster than per-frame seeking).
  • --reuse-images-dir DIR: hardlink already-extracted frame_NNNNNN.* images (zero extra disk) instead of re-decoding the video; falls back to copy/symlink.
  • --min-box-px N / --min-score S: drop tiny or low-confidence boxes.
  • --width / --height: override frame size; --seed: deterministic 70/20/10 split (only affects random).

GUI

uv run python -m vaila.sam_to_yolo

File pickers: SAM tracks CSV → source video → optional reuse-images dir.

Train the resulting dataset

Task auto-detects to detect (no kpt_shape):

uv run python -m vaila.yolotrain --data /path/out_dataset/data.yaml --task detect --model yolo26x.pt --epochs 100 --imgsz 1280 --device 0

After training, track with yolov26track using your best.pt; BoT-SORT assigns the persistent IDs.

Output layout

out_dataset/
├── data.yaml          # nc: 1, names: ['person'], no kpt_shape
├── classes.txt        # person
├── README_dataset.txt # provenance + ready-to-run train command
├── train/images  train/labels
├── val/images    val/labels
└── test/images   test/labels
