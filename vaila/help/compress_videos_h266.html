⚠️ Important Notes

  • H.266/VVC encoding is EXTREMELY SLOW and CPU-intensive
  • No GPU acceleration available for VVC
  • Requires FFmpeg compiled with libvvenc support
  • Standard system FFmpeg usually does NOT include libvvenc

📖 Description

Compresses videos to H.266 (VVC) format using FFmpeg with libvvenc.

  • Parallel processing: speed up batch compression with multiple workers
  • Adaptive compression: automatically discard output files larger than input
  • Encoder check before starting (graceful error if libvvenc missing)
  • QP-based quality (Quantization Parameter 0-63)
  • Resolution control: keep original or downscale
  • Recursive search: process subdirectories (CLI --recursive, GUI subdir depth)
  • Subdir depth: limit recursion depth (0 = only folder, -1 = unlimited)
  • Cross-platform: Windows, Linux, macOS

Getting FFmpeg with libvvenc

  • Windows: gyan.dev full build
  • Linux/macOS: johnvansickle.com static build

Verify: ffmpeg -encoders | grep vvenc

🚀 CLI Usage

# Basic usage
python -m vaila.compress_videos_h266 --dir /path/to/videos

# Recursive (all subdirs)
python -m vaila.compress_videos_h266 --dir /path/to/videos --recursive

# Recursive with max depth 2
python -m vaila.compress_videos_h266 --dir /path/to/videos -r --depth 2

# Custom quality and resolution
python -m vaila.compress_videos_h266 --dir /path/to/videos --preset slow --qp 28 --resolution 1920x1080
OptionDefaultDescription
--dir(required)Directory containing videos
--recursive / -roffRecurse into subdirectories
--depth / -d-1Max subdir depth when recursive (0=root, 1/2/3=levels, -1=unlimited)
--presetmediumultrafast → veryslow
--qp32Quality (0-63). Lower = better
--resolutionoriginale.g. 1920x1080
--workers/-w1Number of parallel workers
