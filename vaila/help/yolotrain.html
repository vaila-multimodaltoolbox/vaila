Description

YOLO26-first trainer for datasets generated by getpixelvideo.py, SAM workflows loaded into getpixelvideo, and existing Ultralytics YOLO datasets.

Auto Task Defaults

  • detect: bbox labels, default yolo26m.pt
  • pose: kpt_shape in data.yaml, default yolo26m-pose.pt
  • segment: polygon labels, default yolo26m-seg.pt

The CLI rejects known model/task mismatches early. For example, a dataset with kpt_shape is YOLO-pose, so --model yolo26x.pt is rejected and --model yolo26x-pose.pt is suggested.

GUI

Main vailá window: YOLO + SAM → Train YOLOv26. Select the dataset folder or data.yaml, keep Task/Model as auto, then start training.

CLI

uv run python -m vaila.yolotrain --data /ABS/path/to/data.yaml --task auto --model auto --dry-run
uv run python -m vaila.yolotrain   --data /ABS/path/to/data.yaml   --task auto --model auto   --epochs 100 --batch 16 --imgsz 640 --device 0   --name yolo_training

Layouts

Supports train/images + train/labels and FIFA/Ultralytics images/train + labels/train layouts.
