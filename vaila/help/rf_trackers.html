GUI: Frame B → Video AI tools → Roboflow trackers (v2.4)

Description

Ultralytics YOLO26 (default: yolo26x.pt in vaila/models/) + Roboflow trackers 2.4.x (SORT, ByteTrack, OC-SORT, BoT-SORT). BoT-SORT CMC: pass the video frame into update (release notes).

CLI progress

By default, the CLI prints a short status block and a tqdm progress bar on stderr. Use --quiet / -q to disable.

uv run python -m vaila.rf_trackers --help

Prints all CLI flags and an epilog with a FIFA sample command.

GUI (no arguments)

uv run python -m vaila.rf_trackers

CLI example (FIFA video)

uv run python -m vaila.rf_trackers \
  -i /path/to/FIFA/to_sent/BRA_KOR_234113.mp4 \
  -w vaila/models/yolo26x.pt \
  --tracker botsort \
  --conf 0.25

Options: --no-save-video, --no-cmc (BoT-SORT), --cmc-method (sparseOptFlow | orb | sift | ecc).

Output

processed_rf_trackers_YYYYMMDD_HHMMSS/ beside the input video:

FilePurpose
rf_tracks.csvLong format (frame, ids, boxes, classes) — debug / analysis
{Label}_id_XX.csvPer-ID vailá/yolov26 header; one row per frame with NaN when missing
all_id_detection.csvWide merge for Get Pixel Video (multi-object tracking CSV)
<stem>_rf_tracked.mp4Optional overlay video

Load all_id_detection.csv or per-ID files in Get Pixel Video. rec2d uses the pixel CSV from Get Pixel Video, not bbox CSVs. Prefer a fresh output folder when re-running the merge.
