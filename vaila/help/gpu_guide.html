Dynamic Hardware Optimization

vailá is designed to adapt intelligently to the hardware it runs on. Whether you are using a high-end workstation with an RTX 4090 or a sleek laptop with limited graphic memory, the toolbox automatically configures itself for the best possible performance.

How it Works

The built-in HardwareManager scans your system at startup to detect OS, GPU Model and VRAM.

Based on this scan, it assigns one of three performance profiles:

Profile Hardware Description Settings
ULTRA > 20GB VRAM High-end (RTX 4090, 3090) FP16, 8GB Workspace
HIGH 7GB - 20GB Mid-range (RTX 5050, 4070) FP16, 2GB Workspace
LITE < 7GB or CPU Entry-level / CPU-only FP32 (CPU) or minimal settings

Auto-Export & Cross-Platform Support

vailá uses TensorRT (.engine files) for extreme acceleration. These files are hardware-specific.

To solve this, vailá implements Auto-Export:

  1. Automatic Detection: Checks if an engine exists for your specific OS and GPU.
  2. Automatic Creation: If missing, it creates it from the source model. Example: yolo26x-pose_NVIDIA_GeForce_RTX_4090.engine
  3. Coexistence: Multiple engines for different machines can exist side-by-side.

⚠️ First Run Notification

The first time you run a new model on a new computer, the process can take 2 to 10 minutes while it builds the engine. Do not close the window. Subsequent runs will be instant.

Windows vs. Linux

The experience is identical. If you dual-boot, vailá will manage engines for both Windows and Linux automatically.
