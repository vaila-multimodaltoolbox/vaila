Description

Deadlift/RDL biomechanical analysis supporting two data types:

  1. IMU data (accelerometer + gyroscope CSV from barbell-mounted sensors)
  2. MediaPipe Pose CSV files with landmark coordinates

The script auto-detects the data type from CSV column headers and routes to the appropriate pipeline.

IMU Pipeline

  • Automatic repetition detection from filtered vertical acceleration
  • Per-rep velocity estimation via numerical integration with drift correction
  • Per-rep power (P = F × v) and work (W = ∫P dt) from barbell weight
  • Cadence metrics (reps/min, inter-rep intervals, timing variability)
  • Center of mass trajectory (barbell displacement)
  • Rep-to-rep comparison: max, min, range, CV%, best/worst rep

MediaPipe Pipeline

  • Multi-repetition detection from averaged wrist/bar-height phases: low-start counts the first standing maximum; standing-start skips setup and counts the next standing maximum after the low phase
  • Whole-body center of mass estimation (De Leva 1996 segment proportions)
  • Foot spread (ankle-to-ankle) and knee spread (knee-to-knee) distances
  • Hand-to-foot horizontal distance
  • Per-rep power & work from COM vertical velocity and configured load mass
  • Cadence and rep-to-rep comparison statistics
  • Deadlift variant classification: stiff-legged, RDL, conventional, or mixed
  • Form quality validations (arm verticality, bar over midfoot, pull synchronism)

Form Validations (MediaPipe)

  • Arm verticality: shoulder-to-wrist horizontal offset at setup; tolerance 5 cm.
  • Bar over midfoot: wrist projection against heel-to-foot-index midpoint; tolerance 3 cm.
  • Initial pull synchronism: first 15% of concentric pull; flags knee extension velocity > 2× hip opening velocity.

Metrics

  • Stance width ratio, foot spread, knee spread
  • Knee flexion, tibia angle, spine deviation, cervical deviation
  • Bar path proximity, hand-to-foot horizontal offset, bar marker height
  • Center of mass trajectory (x, y) and averaged wrist/bar marker trajectory
  • Per-rep: count frame/time, bottom frame/time, peak/mean velocity, peak/mean power, work, COM ROM, bar ROM, load mass
  • Cadence: reps/min, mean/std rep duration, interval variability
  • Rep comparison: max, min, range, CV%, best/worst rep

Usage

GUI: run uv run vaila.py, click Deadlift, choose the data source (Kinematics (MediaPipe) for this module or IMU (AHRS) for the companion), then select a folder containing CSV files. Kinematics mode prints progress in the terminal and opens one centered parameter window with one parameters-file selector and manual values. Click Run to start or Cancel to abort before output folders are created.

uv run python vaila/vaila_deadlift.py -i path/to/data.csv -o path/to/output
uv run python vaila/vaila_deadlift.py -i path/to/data.csv -o path/to/output \
  --config-params tests/Deadlift/kinematics/deadlifit_kinematics_parameters.txt
uv run python vaila/vaila_deadlift.py -i path/to/data.csv -o path/to/output \
  --kinematics-params path/to/kinematics_parameter.txt \
  --subject-params path/to/subject_parameters.txt \
  --fps 59.94 --mass-kg 75 --shank-length-m 0.44 --barbell-mass-kg 20 \
  --cutoff 6 --gif-duration-s 7.2

GIF playback speed

The GIF interval (s) field controls how many seconds each keyframe of the stick-figure animation stays on screen. The current default is 7.2 s (6× slower than the legacy 1.2 s value), which is comfortable for technique review. Click the ? button next to the field for an in-app cheat sheet.

  • Higher value → slower playback (each pose lingers longer)
  • Lower value → faster playback
  • 0.5 s → very fast preview
  • 1.2 s → legacy default (fast)
  • 3.6 s → 3× slower than legacy
  • 7.2 s → current default (6× slower than legacy)
  • 12.0 s → very slow, good for technique review

To make the GIF N times slower than the current default, multiply the value by N (for example, 7.2 × 2 = 14.4 s for 2× slower).

Configuration

Optional vaila_deadlift_config.toml beside the input CSV or in vaila/:

[deadlift_context]
fps = 30
shank_length_m = 0.40
mass_kg = 75
weight_kg = 20
use_total_mass_for_power = true
imu_fps = 25

The GUI uses one parameters-file selector. deadlift_kinematics_parameters.txt can supply camera FPS/Hz plus subject mass, shank length, and bar plus plates mass. Kinematics folders may also use one-row CSV text files such as deadlift_kinematics_parameters.txt or the legacy typo deadlifit_kinematics_parameters.txt:

subject_mass_kg,subject_height_m,shank_m,deadlift_mass_kg,fps_hz
75.0,1.75,0.40,20.0,59.866844999

Outputs

IMU mode

  • Processed time series CSV (vert. acceleration, velocity, displacement)
  • Per-rep summary CSV
  • Time series plot, per-rep bar charts, velocity overlay
  • IMU biomechanical HTML report

MediaPipe mode

  • Kinematics CSV with frame-by-frame metrics
  • Calibrated CSV with marker, COM, and bar-height columns
  • Per-rep summary CSV
  • Lower limb, safety, body distances with time axes and peak/valley markers, per-rep, stick-figure, and labeled low/standing keyframe GIF presentation outputs
  • Biomechanical HTML report with variant classification
