üìñ Description

Advanced version of the 2D pose estimation script that incorporates YOLOv11 for person detection to improve MediaPipe's performance. It offers robust multi-person tracking, advanced temporal filtering, and detailed reporting.

Key Features:

  • YOLOv11 Integration: Uses YOLO models (n, s, m, l, x) for accurate person detection.
  • Dual Modes: yolo_only (17 keypoints) and yolo_mediapipe (33 landmarks).
  • Advanced Filtering: Moving Median (outlier removal), Kalman (tracking), and Savitzky-Golay (smoothing).
  • Comprehensive Output: Annotated video, normalized CSV, pixel CSV, YOLO raw CSV, and detailed reports.

‚öôÔ∏è Configuration & GUI

The main configuration dialog now includes easy-to-use buttons for managing your settings:

  • Save Config: Save current parameters to a .toml file.
  • Load Config: Load parameters from a saved .toml file.
  • Create Config: Reset all parameters to default values.
  • Help: Open this documentation.

Parameters

  • Filter Type:
    • none: Raw data.
    • median: Removes short-term outliers/spikes (Window=5).
    • kalman: Predictive filter, good for continuous tracking.
    • savgol: Polynomial smoothing.
  • YOLO Mode: Choose between YOLO-only (faster) or Hybrid (more detailed).
  • Model Complexity: MediaPipe model size (0, 1, 2).

üìÅ Output Files

For each processed video, the following files are generated:

  • *_mp.mp4: Annotated video with landmarks and skeleton.
  • *_mp_norm.csv: Normalized coordinates (0-1).
  • *_mp_pixel.csv: Pixel coordinates mapped to video resolution.
  • *_yolo_pixel.csv: Raw YOLO keypoints (if available).
  • config.toml: A copy of the configuration used.
  • report.txt: Detailed processing log and metadata.

üîß Main Functions

  • process_video: Main processing pipeline.
  • detect_persons_with_yolo: Handles YOLO inference.
  • apply_median_filter: Applies moving median filter.
  • apply_kalman_filter / apply_savgol_filter: Other temporal filters.
  • download_or_load_yolo_model: Manages model assets.
