Overview

EMG (Electromyography) analysis toolkit. Computes time-, frequency-, and time-frequency-domain features, produces CSV results, PNG/SVG figures, a statistical summary and an optional HTML report. The CSV reader auto-detects multiple acquisition-system layouts.

Supported CSV input formats

1. Standard CSV (Anglo-Saxon decimal)

time,EMG_volts
0.0000,0.000123
0.0005,-0.000087

. decimal, , field separator. The first column containing EMG or Volt in its name is auto-selected.

2. Trigno / Delsys EMGworks (European decimal)

X_s,Trigno_sensor_14:_EMG_14_Volts,X_s,Trigno_sensor_14:_Acc_14.X_g,...
0,00000e+000,0,00000e+000,0,00000e+000,0,00000e+000,...
5,19231e-004,0,00000e+000,6,75000e-003,0,00000e+000,...

The comma is both decimal separator and CSV separator. The header has N fields but every data row has 2N fields. The reader recombines field pairs ("5" + "19231e-004" → "5.19231e-004") and re-parses the buffer. Interleaved EMG + accelerometer channels are supported: only the EMG column is auto-picked.

3. Generic multi-column CSV

Pass --channel <name|substring|index> to force a specific column.

Sample rate auto-detection

fs = 1 / median(Δt) computed from the time column. Override with --fs <Hz>.

SystemTypical fs (Hz)
Trigno EMG1925.93
EMGworks Delsys2000
Biopac1000 – 2000
Noraxon1500 – 2000

CLI usage

# GUI (no arguments)
uv run vaila/emg_labiocom.py

# Inspect a file
uv run vaila/emg_labiocom.py --inspect /path/to/file.csv

# Single file, full signal, headless, with HTML report
uv run vaila/emg_labiocom.py \
    -i /home/preto/Desktop/aula_fim/emg_aula_26.csv \
    -o ./results --full --no-plot --report

# Batch, force fs=2000 Hz, force channel by substring
uv run vaila/emg_labiocom.py -i ./emg_dir -o ./out --fs 2000 -c EMG_14 --full

# Explicit segments (sample indices)
uv run vaila/emg_labiocom.py -i file.csv -o out -s "1000,5000;10000,15000"

# Data already in microVolts
uv run vaila/emg_labiocom.py -i file.csv -o out --unit-scale 1.0 --full

CLI flags

FlagPurpose
-i, --inputInput CSV/TXT file or directory of CSV/TXT files.
-o, --outputOutput directory (a emg_labiocom_<timestamp>/ is created inside).
--fsSample rate (Hz). Defaults to auto-detected.
-c, --channelColumn name, substring (case-insensitive), or 0-based index.
-s, --selectionsExplicit segments "start1,end1;start2,end2;...".
--fullAnalyse the whole signal as one segment.
--reportRender HTML report per file + index.html.
--no-plotHeadless: no interactive plots (PNG/SVG still saved).
--unit-scaleSignal multiplier (default 1e6, i.e. V → µV; use 1.0 if already µV).
--inspect FILEPrint detected layout for FILE and exit.
--guiForce GUI mode.

GUI usage

Launch from vaila.py → Frame B → EMG Analysis, or directly:

uv run vaila/emg_labiocom.py
  1. Select an input directory containing CSV/TXT files.
  2. Select a reference EMG file for segmentation.
  3. The reader auto-detects layout and pre-fills the EMG channel and sample rate.
  4. Confirm or change the EMG channel (substring match).
  5. Confirm or change the sample rate.
  6. Choose Interactive (mouse) or Manual selection mode.
  7. Choose whether to show plots and whether to generate HTML reports.
  8. Select the output directory.

Analyses computed per segment

  • Band-pass filter (Butterworth 4th-order, 10 – 450 Hz)
  • Full-wave rectification + linear envelope (10 Hz low-pass)
  • RMS (250 ms window, 50 % overlap) with polynomial fit
  • Median frequency (Welch PSD) with polynomial fit
  • Power spectral density (Welch) with peak frequency
  • Spectrogram (STFT)
  • Continuous wavelet transform (Morlet) — when PyWavelets is installed
  • Time-domain features: ZCR, MAV, VAR, WL, Wilson amplitude, slope-sign changes
  • Frequency-domain features: mean / peak frequency, spectral variance, band powers, power ratio, MMFP
  • Fatigue indices: classic FI, modified FI, Dimitrov FI, MF slope
  • Statistical features: skewness, kurtosis, Hjorth mobility & complexity

Programmatic API

from vaila.emg_labiocom import (
    _read_emg_csv_smart,     # raw DataFrame loader
    _select_emg_channel,     # (time, emg, t_col, e_col, fs_est)
    _load_emg_signal,        # (emg_microvolts, fs_est, e_col)
    emg_analysis,            # full pipeline on one file
    run_emg_cli,             # argparse entry point
    run_emg_gui,             # Tkinter entry point
)

df = _read_emg_csv_smart("emg_aula_26.csv")
time, emg, t_col, e_col, fs = _select_emg_channel(df)
print(t_col, e_col, fs)
