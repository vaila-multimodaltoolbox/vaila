soccerfield_calib — DLT2D homography from soccer-field keypoints

Module: vaila/soccerfield_calib.py
GUI button: Soccer-Field Calib in Frame B
CLI: uv run vaila/soccerfield_calib.py …
Reference: vaila/models/soccerfield_ref3d.csv (29 FIFA keypoints, 105 × 68 m)

What it does

Fits a DLT2D homography (8 coefficients) that maps a fixed broadcast frame to the FIFA 105 × 68 m ground plane (Z = 0), using ≥ 6 manually clicked keypoints. The output .dlt2d is compatible with vaila/rec2d.py.

This is a single-frame / static-camera tool. If the camera moves (typical broadcast), use fifa_to_dlt.py (button FIFA cams→DLT) for per-frame DLT instead.

Step-by-step (GUI)

  1. Open vailá and click Soccer-Field Calib in Frame B.
  2. Pick the broadcast video / fixed frame.
  3. getpixelvideo opens — click at least 6 named keypoints (corners, midfield, penalty spots …).
  4. Save. The script fits the homography and writes <stem>.dlt2d + report.
  5. If a FIFA --data-root is set, you also get cameras/<stem>_homography.npz.

CLI quick recipes

Recipe 1 — Static frame, click in GUI

uv run vaila/soccerfield_calib.py -v /path/to/video.mp4 -o /path/to/out

Recipe 2 — Use a pre-clicked pixel CSV

uv run vaila/soccerfield_calib.py \
  -v /path/to/video.mp4 -p /path/to/pixels.csv \
  --frame 0 -o /path/to/out

Recipe 3 — FIFA fallback (no official cameras NPZ)

uv run vaila/soccerfield_calib.py \
  -v /path/to/data/videos/SEQ.mp4 \
  --data-root /path/to/data \
  -o /path/to/data/cameras

Produces cameras/SEQ_homography.npz.

Recipe 4 — Restrict keypoint set

uv run vaila/soccerfield_calib.py -v video.mp4 -o out/ \
  --keypoints bottom_left_corner,bottom_right_corner,top_left_corner,top_right_corner,center_field,left_penalty_spot

Recipe 5 — From a SAM3 batch output

uv run vaila/soccerfield_calib.py \
  --from-sam /path/to/processed_sam_<ts>/<video>/ \
  --pixels /path/to/clicks.csv

Recipe 6 — List valid keypoint names

uv run vaila/soccerfield_calib.py --list-keypoints

Pixel CSV format

  • Long: name,x,y — names must match soccerfield_ref3d.csv.
  • Wide: frame,p1_x,p1_y,p2_x,p2_y,… (getpixelvideo layout). Combine with --frame N.

Both are produced by vaila.soccerfield_keypoints_ai (field_keypoints_getpixelvideo.csv for the wide one).

Output files

<output>/
  <stem>_ref2d.csv                  # world XY for fitting
  <stem>.dlt2d                      # 8 DLT coefficients (1 row)
  <stem>_homography_report.txt      # per-point error (px, m)
  cameras/<stem>_homography.npz     # only when --data-root is set

End-to-end with the AI seed

# 1. AI seed (32 keypoints)
uv run python -m vaila.soccerfield_keypoints_ai \
  --mode video -i video.mp4 -o out_kps/ \
  --backend ultralytics \
  --weights vaila/models/runs/pose_fifa/pitch32_recipeA_400ep/weights/best.pt \
  --imgsz 1280 --conf 0.30 --draw-min-conf 0.40 --device 0 \
  --stride 1 --overlay-video

# 2. (optional) refine clicks in getpixelvideo

# 3. DLT2D homography (one frame)
uv run vaila/soccerfield_calib.py \
  -v video.mp4 \
  -p out_kps/processed_field_kps_*/field_keypoints_getpixelvideo.csv \
  --frame 0 -o out_calib/

# 4. Player pixel → field metres
uv run vaila/rec2d.py \
  --dlt-file out_calib/video.dlt2d \
  --input-dir player_pixels/ \
  --output-dir player_world/ --rate 30

For moving cameras, replace step 3 by FIFA cameras/*.npz → per-frame DLT — see vaila_sam.html (section Full broadcast pipeline).
