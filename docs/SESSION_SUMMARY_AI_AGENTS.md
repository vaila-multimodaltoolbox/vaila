## What Was Fixed

### `.gitignore` — two critical errors were found and corrected

**Error 1 — `uv.lock` was being ignored:**
```gitignore
# BEFORE (wrong)
uv.lock   ← was in .gitignore

# AFTER (correct)
# uv.lock → do NOT ignore — must be committed for reproducible builds
```
The `uv.lock` file is the equivalent of `package-lock.json`. It must be committed so that `uv sync --frozen` produces identical environments on every machine and in CI.

**Error 2 — `.python-version` was being ignored:**
```gitignore
# BEFORE (wrong)
.python-version   ← was in .gitignore

# AFTER (correct)
# .python-version → do NOT ignore — generated by `uv python pin 3.12`
```
This file pins the Python version for the project. Without it, `uv` may use a different Python version on another machine.

**Minor fix — duplicate `*.AVI` entry removed.**

```gitignore
# Track Cursor rules, ignore Cursor local config

# Track CLAUDE.md (root), ignore local/private Claude config
```

These exception patterns (`!`) are the correct way to selectively track files inside otherwise-ignored directories.

## The Astral Toolchain

The entire project uses tools from https://astral.sh — all written in Rust, extremely fast:

| Tool | Role | Replaces |
|------|------|---------|
| `uv` | Package manager, venv, Python installer | pip, poetry, pyenv, virtualenv |
| `ruff` | Linter + formatter | flake8, black, isort, pyupgrade |
| `ty` | Type checker (beta) | mypy, Pyright |

**Critical rule:** never use `pip install`, `black`, `isort`, `flake8`, or `mypy` in this project. Always use `uv run ruff` and `uv run ty`.

**Full QA pipeline before every commit:**
```bash
uv run ruff check vaila/ --fix
uv run ruff format vaila/
uv run ty check vaila/
uv run pytest tests/ -v
```

## What You Should Know If You Work on This Project

1. **Always use `uv run`** — never activate the venv manually or use bare `pip`
2. **Every `vaila/` module** must use the dual-import pattern (relative import with absolute fallback)
3. **Tkinter only** for GUI — no Qt, wx, or other frameworks
4. **Scientific variable names** (X, Y, Z, F, R, T) are valid and ruff is configured to accept them
5. **Output directories** are always timestamped: `processed_<type>_YYYYMMDD_HHMMSS/`
6. **`hardware_manager.py`** handles all GPU/CPU detection — never duplicate this logic
7. **`ty` is in beta** — use it as a complement to ruff, not as a strict hard gate
8. **No files ≥20 MiB** — the pre-commit hook installed by `bash install-hooks.sh` enforces this
