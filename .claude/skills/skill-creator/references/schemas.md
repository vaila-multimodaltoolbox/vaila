# Skill-Creator JSON Schemas

All JSON file structures used by the skill-creation workflow.

---

## `evals/evals.json`

Root of the evals file — defines all test cases for a skill.

```json
{
  "skill_name": "my-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "User's task prompt",
      "expected_output": "Description of expected result",
      "files": [],
      "assertions": [
        {
          "id": "a1",
          "text": "Assertion description (descriptive name readable in viewer)",
          "check": "how to verify this assertion",
          "type": "file_exists | string_contains | output_format | programmatic"
        }
      ]
    }
  ]
}
```

### Fields

| Field | Type | Description |
|---|---|---|
| `skill_name` | string | Name of the skill being evaluated |
| `evals[].id` | integer | Unique eval ID |
| `evals[].prompt` | string | The user's task prompt (what to test) |
| `evals[].expected_output` | string | Human-readable description of expected result |
| `evals[].files` | array | Input files needed for the test (paths relative to workspace) |
| `evals[].assertions[].id` | string | Unique assertion ID within eval |
| `evals[].assertions[].text` | string | Descriptive name shown in benchmark viewer |
| `evals[].assertions[].check` | string | How to verify — file path, string to find, format description, or programmatic script |
| `evals[].assertions[].type` | string | `file_exists`, `string_contains`, `output_format`, or `programmatic` |

### Assertion Types

- **`file_exists`**: `check` is a file path. PASS if file exists, FAIL otherwise.
- **`string_contains`**: `check` is a substring. PASS if found in output, FAIL otherwise.
- **`output_format`**: `check` describes expected format. PASS if output conforms, FAIL otherwise.
- **`programmatic`**: `check` is a path to a Python script that exits 0 for pass, non-zero for fail. The script receives the output directory as `$1`.

## `eval_metadata.json`

Per-eval metadata written alongside each run directory.

```json
{
  "eval_id": 0,
  "eval_name": "descriptive-name-here",
  "prompt": "The user's task prompt",
  "assertions": [
    {
      "id": "a1",
      "text": "Assertion description",
      "check": "how to verify",
      "type": "file_exists"
    }
  ]
}
```

Copied from `evals/evals.json` at eval time, with assertions populated.

## `grading.json`

Results of evaluating assertions against a run's output.

```json
{
  "expectations": [
    {
      "text": "Assertion description (same as in evals)",
      "passed": true,
      "evidence": "Why it passed or failed — e.g., found 'foo' in output.txt, or file exists at /path/to/result.csv"
    },
    {
      "text": "Another assertion",
      "passed": false,
      "evidence": "Expected 'bar' but output contained only 'baz'"
    }
  ]
}
```

**Important**: Must use `text`, `passed`, `evidence` — not `name`/`met`/`details` or other variants. The eval-viewer expects these exact fields.

### Pass/fail rules by type

| Type | PASS if | FAIL if |
|---|---|---|
| `file_exists` | File exists at path | File missing |
| `string_contains` | Substring found in output | Substring not found |
| `output_format` | Output conforms to format | Output malformed |
| `programmatic` | Script exits 0 | Script exits non-zero |

## `benchmark.json`

Aggregated benchmark results for one iteration. Generated by `scripts/aggregate_benchmark.py`.

```json
{
  "skill_name": "my-skill",
  "iteration": 1,
  "configurations": [
    {
      "name": "with_skill",
      "pass_rate": 0.83,
      "mean_time_seconds": 45.2,
      "mean_tokens": 82000,
      "stddev_time": 8.3,
      "stddev_tokens": 12000,
      "evals": [
        {
          "eval_name": "basic-csv-processing",
          "pass_rate": 1.0,
          "time_seconds": 38.1,
          "tokens": 71000,
          "passed": true
        },
        {
          "eval_name": "complex-xlsx-analysis",
          "pass_rate": 0.5,
          "time_seconds": 52.3,
          "tokens": 93000,
          "passed": false
        }
      ]
    },
    {
      "name": "without_skill",
      "pass_rate": 0.33,
      "mean_time_seconds": 61.4,
      "mean_tokens": 95000,
      "stddev_time": 12.1,
      "stddev_tokens": 18000,
      "evals": [
        {
          "eval_name": "basic-csv-processing",
          "pass_rate": 0.0,
          "time_seconds": 55.2,
          "tokens": 88000,
          "passed": false
        },
        {
          "eval_name": "complex-xlsx-analysis",
          "pass_rate": 0.5,
          "time_seconds": 67.6,
          "tokens": 102000,
          "passed": false
        }
      ]
    },
    {
      "name": "delta",
      "pass_rate": 0.50,
      "mean_time_seconds": -16.2,
      "mean_tokens": -13000,
      "stddev_time": null,
      "stddev_tokens": null,
      "evals": []
    }
  ],
  "analyst_notes": [
    "Assertion 'output_contains_timestamp' passes on both with_skill and without_skill — non-discriminating.",
    "Eval 'complex-xlsx-analysis' has high time variance (stddev 15.2s) — consider running multiple trials."
  ],
  "timestamp": "2025-01-15T10:30:00Z"
}
```

### Fields

| Field | Type | Description |
|---|---|---|
| `configurations[].name` | string | `with_skill`, `without_skill`, or `delta` |
| `configurations[].pass_rate` | float | Fraction of assertions that passed (0-1) |
| `configurations[].mean_time_seconds` | float | Average wall-clock time across evals |
| `configurations[].mean_tokens` | float | Average token count across evals |
| `configurations[].stddev_time` | float | Standard deviation of time |
| `configurations[].stddev_tokens` | float | Standard deviation of tokens |
| `configurations[].evals[].pass_rate` | float | Per-eval pass rate |
| `configurations[].evals[].passed` | bool | Overall eval pass (any assertion failing = false) |
| `configurations[].evals[].time_seconds` | float | Time for this eval |
| `configurations[].evals[].tokens` | float | Tokens for this eval |
| `delta.mean_time_seconds` | float | with_skill minus without_skill (negative = faster) |
| `delta.mean_tokens` | float | with_skill minus without_skill (negative = fewer tokens) |
| `analyst_notes[]` | string | Auto-generated notes from analyzer agent |

## `timing.json`

Captured immediately when a subagent task completes.

```json
{
  "total_tokens": 84852,
  "duration_ms": 23332,
  "total_duration_seconds": 23.3
}
```

Written to the run directory immediately on task completion notification — this is the only chance to capture it.

## `feedback.json`

Exported from the eval-viewer when the user clicks "Submit All Reviews."

```json
{
  "reviews": [
    {
      "run_id": "eval-0-with_skill",
      "feedback": "the chart is missing axis labels",
      "timestamp": "2025-01-15T10:35:00Z"
    },
    {
      "run_id": "eval-1-with_skill",
      "feedback": "",
      "timestamp": "2025-01-15T10:35:05Z"
    },
    {
      "run_id": "eval-2-with_skill",
      "feedback": "perfect, love this",
      "timestamp": "2025-01-15T10:35:10Z"
    }
  ],
  "status": "complete",
  "iteration": 1,
  "submitted_at": "2025-01-15T10:35:15Z"
}
```

Empty `feedback` means the user thought it was fine. Focus improvements on cases with specific complaints.

## `trigger_eval.json`

Used by the description optimizer. Saved after user review via `assets/eval_review.html`.

```json
[
  {
    "query": "ok so my boss just sent me this xlsx file (its in my downloads, called something like 'Q4 sales final FINAL v2.xlsx') and she wants me to add a column that shows the profit margin as a percentage. The revenue is in column C and costs are in column D i think",
    "should_trigger": true,
    "notes": "casual speech, real filename context"
  },
  {
    "query": "create a summary report from these numbers",
    "should_trigger": true,
    "notes": "ambiguous but clearly needs spreadsheet skill"
  },
  {
    "query": "write a fibonacci function in python",
    "should_trigger": false,
    "notes": "clearly not about spreadsheets"
  }
]
```

### Fields

| Field | Type | Description |
|---|---|---|
| `query` | string | Realistic user prompt |
| `should_trigger` | bool | Whether the skill should trigger for this query |
| `notes` | string | Optional explanation of why this is a good/bad test |

Queries should be substantive (complex, multi-step) not trivial. See SKILL.md "How skill triggering works" section.
